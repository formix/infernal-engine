done(erropt)

The done function tells the inference engine that the current rule is terminated and that the next rule shall be executed. If no more rule are in the agenda, the inferenceCallback function is called without any parameter. If the done function is called with a parameter, the inference immediately stops (no more rules are executed from the agenda) and the inferenceCallback is called with the same parameter (the error).
Parameters:
Name Type Attributes Description
err Error | * <optional>
 The error information to send to the inferenceCallback

rule(done)

The rule callback is a function that takes a single done callback method. This method is executed in the context of an EngineProxy instance. It is important to note that before exiting a rule function, the done callback function has to be called to inform the engine that the next rule in the agenda has to be executed. Usually after calling the done function the method should exits by explicitely calling 'return' or by letting the execution exit the scope of the function.

traceCallback(trace, trace#action, trace#ruleopt, trace#factopt, trace#oldValueopt, trace#newValueopt, trace#messageopt)

The trace callback is called when an event changing the engine state is hapenning. See InfernalEngine#startTracing for details.
Parameters:
Name Type Attributes Description
trace object The trace data.
trace#action string Can be either 'reset', 'set', 'notify', 'addRule', 'infer' or 'trace'.
trace#rule string <optional>
 The rule name that generated the trace. This property is undefined if the trace was not generated during inference.
trace#fact string <optional>
 The fact name if the trace action is 'set' or 'notify'.
trace#oldValue * <optional>
 The previous value of the fact if the trace action is 'set'.
trace#newValue * <optional>
 The new value of the fact if trace action is 'set' or 'notify'.
trace#message string <optional>
 The message sent by the {@ling EngineProxy#trace} method if trace action is 'trace'.
